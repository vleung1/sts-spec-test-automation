# CTDC term-by-value verification artifacts

Generated by **`python tests/term_verify/ctdc_term_verify.py`** (port of `verify_ctdc_term_endpoint_from_sheet.py` + enrich/extract).

**Source YAML:** `data/data-models-yaml/ctdc_model_properties_file-2.yaml` (same as `extract_ctdc_enum_properties.py` in `termValue_verification_scripts`).

| File | Description |
|------|-------------|
| `ctdc_enum_properties_summary.csv` | One row per property with an `Enum` block |
| `ctdc_enum_terms_for_verification.csv` | One row per `(prop_handle, enum_value)` (term handle from YAML) |
| `ctdc_enum_terms_for_verification_enriched.csv` | Plus STS version, node, API **term_value** |
| `ctdc_term_endpoint_verification_report.csv` | Per-row HTTP result for `GET .../term/{termValue}` |
| `ctdc_term_endpoint_verification_report.md` | Summary + failed rows (preview) |

## Regenerate

```bash
pip install -e .
python tests/term_verify/ctdc_term_verify.py
```

Options: `--yaml`, `--out-dir`, `--base-url`, `--limit N`, `--skip-extract`, `--skip-enrich`, `--warn-only`.

Uses paginated `/terms`, inline `#` comment stripping on enum lines, and verifies with **term_value** only (not handle).
