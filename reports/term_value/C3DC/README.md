# C3DC term-by-value verification artifacts

Generated by **`python tests/term_verify/c3dc_term_verify.py`**, same behavior as [../CCDI/README.md](../CCDI/README.md).

| File | Description |
|------|-------------|
| `c3dc_enum_properties_summary.csv` | One row per YAML property with an `Enum` block |
| `c3dc_enum_terms_for_verification.csv` | One row per `(prop_handle, enum_value)` (API term **handle**) |
| `c3dc_enum_terms_for_verification_enriched.csv` | Plus STS `version_string`, `node_handle`, API **term_value** |
| `c3dc_term_endpoint_verification_report.csv` | Per-row result for `GET .../term/{termValue}` |
| `c3dc_term_endpoint_verification_report.md` | Summary + failed rows (preview) |

## Regenerate

```bash
pip install -e .
python tests/term_verify/c3dc_term_verify.py
```

Options: `--yaml`, `--out-dir`, `--base-url`, `--limit N`, `--skip-extract`, `--skip-enrich`, `--warn-only`.

C3DC extract uses flexible `Enum:` indentation (`extract_c3dc_enum_properties` regex), inline `#` comment stripping, and paginated `/terms` during enrich.
