# CCDI-DCC term-by-value verification artifacts

Generated by **`python tests/term_verify/ccdi_dcc_term_verify.py`** (ports `extract_ccdi_dcc_enum_properties.py`, `enrich_ccdi_dcc_query_sheet.py`, and `verify_ccdi_dcc_term_endpoint_from_sheet.py`).

**Source YAML:** `data/data-models-yaml/ccdi-dcc-model-props-3.yml`.

- **Extract** may **HTTP-fetch** remote YAML when an enum entry is an `http(s)` URL (CBIIT property_yamls), then parse `PropDefinitions` lists—same as legacy.
- **Enrich** discovers model handle **`CCDI-DCC`** or **`CCDI_DCC`**, maps properties to nodes, and fills **`term_value`** via paginated **`/terms`** (handle → API value).
- **Verify** uses **`(term_value or '') or (enum_value or '')`** for the URL path (legacy); response must include a term whose **`value`** equals that string.

| File | Description |
|------|-------------|
| `ccdi_dcc_enum_properties_summary.csv` | One row per property with an `Enum` block |
| `ccdi_dcc_enum_terms_for_verification.csv` | One row per `(prop_handle, enum_value)` |
| `ccdi_dcc_enum_terms_for_verification_enriched.csv` | Plus model version, node, API **term_value** |
| `ccdi_dcc_term_endpoint_verification_report.csv` | Per-row HTTP result |
| `ccdi_dcc_term_endpoint_verification_report.md` | Summary + failed rows (preview) |
