# CDS term endpoint verification artifacts

Generated by **`python tests/term_verify/cds_term_verify.py`** (port of `extract_cds_enum_properties.py`, `enrich_cds_query_sheet.py`, and `verify_term_endpoint_from_sheet.py`).

**Source YAML:** `data/data-models-yaml/cds-model-props-4.yml`.

## How CDS differs from CCDI / ICDC / etc.

- **Enrich** only discovers CDS latest version and maps each `prop_handle` to a **node** (first occurrence). There is **no** paginated `/terms` call and **no** `term_value` column.
- **Verify** calls `GET .../term/{termValue}` with **`termValue` = YAML `enum_value`**, and checks the JSON array for an item with `value == enum_value` (same as legacy).

| File | Description |
|------|-------------|
| `cds_enum_properties_summary.csv` | One row per property with an `Enum` block |
| `cds_enum_terms_for_verification.csv` | One row per `(prop_handle, enum_value)` |
| `cds_enum_terms_for_verification_enriched.csv` | Plus `model_handle`, `version_string`, `node_handle` |
| `cds_term_endpoint_verification_report.csv` | Per-row HTTP result (`prop_handle`, `enum_value`, `http_status`, `passed`, `notes`) |
| `cds_term_endpoint_verification_report.md` | Summary + failed rows (preview) |

Row counts can be large (~9k+); use `--limit N` for a quick check.
