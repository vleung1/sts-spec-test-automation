# STS v2 API Test Framework Agent

OpenAPI-driven end-to-end API test automation framework for the **Simple Terminology Server (STS) v2** API. Tests are generated from the OpenAPI spec (`spec/v2.yaml`) and executed against a live STS instance, with optional contract validation and HTML/JSON reporting.

## Purpose

- **Endpoint coverage**: All v2 paths get at least one positive and, where applicable, one negative test.
- **Spec-driven**: Tests are derived from the spec so that spec changes can be reflected by re-running the generator.
- **Functional + negative**: Validates status codes and basic response shape; includes 404/422 negative cases.
- **CI-ready**: Run via the three main scripts (or one-shot `run_full_suite.sh`), optional pytest, or the CLI; configurable base URL and report output.

## Quick Start

### Install

```bash
cd mdb/sts-test-framework-agent
pip install -e .
```

### Configure (optional)

The framework reads **environment variables** (no `.env` auto-load). Primary variable:

- **`STS_BASE_URL`** – STS v2 API root including `/v2` (default: `https://sts-qa.cancer.gov/v2`), resolved by [`sts_test_framework.config.sts_base_url()`](src/sts_test_framework/config.py). Used by pytest, the CLI, and the scripts under `scripts/`.
- **`CADSR_BASE_URL`** / **`CADSR_DESIGNATION_TYPES`** – Used by manual modules `tests/test_manual/test_cadsr_alternatevalues_draftnew_cdes.py` and `tests/test_manual/test_cadsr_multi_concept_cdes.py` (caDSR `DataElement` vs STS PVS: marker **`cadsr_alt_pvs`** for Designations; **`cadsr_draft_new`** checks DRAFT NEW CDEs vs **cde-pvs** (exact **`longName`** vs **`CDEFullName`**, PV subset on **`ncit_concept_code`** rows) and optionally vs **model-pvs** when **`model`** / **`model_version`** / **`property`** are set in the case JSON (PV subset only for model-pvs); **`cadsr_multi_concept_pv`** covers **`case_type`** **`multi_concept_pv`** (multi-Concept PVs) and **`url_pv_yaml_enum_model_pvs`** (caDSR PV value is a URL; STS **cde-pvs** not asserted; **model-pvs** matches vendored YAML **Enum** multiset with all rows null **`ncit_concept_code`**, empty **`synonyms`**) — see `data/cadsr_multi_concept_cdes_cases.json`; ignores **`CADSR_DESIGNATION_TYPES`** for the latter). By default **all** designation names are checked for **`cadsr_alt_pvs`**; set **`CADSR_DESIGNATION_TYPES`** to a comma-separated list of ``Designations[].type`` values to narrow. See [docs/ONBOARDING.md §6.2](docs/ONBOARDING.md#62-configuration-environment-variables).

Also common:

- **`STS_SSL_VERIFY`** – Set to `false` to disable SSL verification (dev/self-signed only).
- **`REPORT_DIR`** – Where the **CLI** writes HTML/JSON reports (default: `reports/`).
- **`STS_PARALLEL_WORKERS`** – Max models to run concurrently in `run_autogenerated_tests.py` (default: `1` sequential; set `2`, `8`, etc. for parallel).
- **`STS_TERM_VERIFY_WORKERS`** – Max `*_term_verify.py` scripts to run at once in `run_all_term_verify.sh` (default: `2`; set to `1` for sequential).

Full variable list (`STS_MODELS`, `STS_DEDUP_LIMIT`, etc.): [docs/ONBOARDING.md §6.2](docs/ONBOARDING.md#62-configuration-environment-variables).

**Tip:** When you run manual tests via `run_manual_tests.sh` (pytest), the session header includes `STS environment: https://...` — confirm that URL before long runs.

### Run tests (recommended: three scripts + optional one-shot)

From the project root after `pip install -e .`. Set `STS_BASE_URL` if not using default QA. See **[docs/RUNBOOK.md](docs/RUNBOOK.md)** for the shortest checklist.

**All three pipelines in order** (manual pytest HTML → per-model OpenAPI HTML → term-verify):

```bash
bash scripts/run_full_suite.sh
```

**Or run each separately:**

```bash
bash scripts/run_manual_tests.sh
python scripts/run_autogenerated_tests.py
bash scripts/run_all_term_verify.sh
```

- **`run_manual_tests.sh`** — `tests/test_manual/` with pytest-html → `reports/manual_tests.html` (extra args go to pytest, e.g. `-m nullcde`).
- **`run_autogenerated_tests.py`** — same OpenAPI functional pipeline as `tests/test_generated/`, but **CLI** timestamped HTML/JSON under `reports/<model>/`. Runs models **one at a time** by default; set `STS_PARALLEL_WORKERS` for parallel. Optional: `STS_MODELS=PSDC,CTDC`.
- **`run_all_term_verify.sh`** — all `tests/term_verify/*_term_verify.py` (default **2** at a time via `STS_TERM_VERIFY_WORKERS`); outputs under `reports/term_value/<COMMONS>/`.

**Via CLI** (single model or one-off report directory):

```bash
python -m sts_test_framework.cli --spec spec/v2.yaml --base-url https://sts-qa.cancer.gov/v2 --report reports/
```

**Test a specific data model** (e.g. PSDC, CTDC) and use the **latest release version** (no pre-release hash):

```bash
STS_BASE_URL=https://sts-qa.cancer.gov/v2 python -m sts_test_framework.cli --report reports/ --model PSDC --release
```

- `--model` – Model handle to test (e.g. `PSDC`, `C3DC`). If omitted, the first model from `/models/` is used.
- `--release` – Use the latest **release** version (version string with no hyphen, e.g. `2.1.0`) from `/model/{handle}/versions`. If omitted, the first version in the list is used. If the model has no release versions, the first available version (e.g. pre-release) is used.

Filter by tags:

```bash
python -m sts_test_framework.cli --tags id,model,models
```

Skip negative cases:

```bash
python -m sts_test_framework.cli --no-negative
```

Use minimal output (e.g. for CI):

```bash
python -m sts_test_framework.cli --quiet --report reports/
```

**Optional: full pytest** (generated + manual + unit in one process; use when you want IDE/pytest workflows rather than the per-model HTML reports): [docs/ONBOARDING.md §6.4](docs/ONBOARDING.md#64-run-with-pytest-recommended-for-day-to-day-work).

```bash
pytest tests/ -v
```

### Term-by-value verification (YAML → enrich → verify)

Ports of `termValue_verification_scripts`: vendored property YAML → extract enums → enrich via STS → verify `GET .../term/{termValue}`. **Most models** use paginated `/terms` to map YAML handle→API **value** for the URL. **CDS** uses YAML **enum_value** in the URL (no `/terms` step); see [data/data-models-yaml/README.md](data/data-models-yaml/README.md).


| Commons  | Script                                             | YAML                                                      | Output                         |
| -------- | -------------------------------------------------- | --------------------------------------------------------- | ------------------------------ |
| CCDI     | `tests/term_verify/ccdi_term_verify.py`            | `data/data-models-yaml/ccdi-model-props.yml`              | `reports/term_value/CCDI/`     |
| C3DC     | `tests/term_verify/c3dc_term_verify.py`            | `data/data-models-yaml/c3dc-model-props.yml`              | `reports/term_value/C3DC/`     |
| CTDC     | `tests/term_verify/ctdc_term_verify.py`            | `data/data-models-yaml/ctdc_model_properties_file-2.yaml` | `reports/term_value/CTDC/`     |
| ICDC     | `tests/term_verify/icdc_term_verify.py`            | `data/data-models-yaml/icdc-model-props.yml`              | `reports/term_value/ICDC/`     |
| CDS      | `tests/term_verify/cds_term_verify.py`             | `data/data-models-yaml/cds-model-props-4.yml`             | `reports/term_value/CDS/`      |
| CCDI-DCC | `tests/term_verify/ccdi_dcc_term_verify.py`        | `data/data-models-yaml/ccdi-dcc-model-props-3.yml`        | `reports/term_value/CCDI-DCC/` |

Run from the project root (requires `pip install -e .` for `sts_test_framework` imports):

```bash
python tests/term_verify/ccdi_term_verify.py
python tests/term_verify/c3dc_term_verify.py
python tests/term_verify/ctdc_term_verify.py
python tests/term_verify/icdc_term_verify.py
python tests/term_verify/cds_term_verify.py
python tests/term_verify/ccdi_dcc_term_verify.py
```

- `**--limit N**` — verify only N rows (quick check).
- `**--warn-only**` — exit 0 even if some rows fail (reports still list failures).
- `**STS_BASE_URL**` — same as other tools (default QA).

See [data/data-models-yaml/README.md](data/data-models-yaml/README.md) and [reports/term_value/README.md](reports/term_value/README.md) (per-commons outputs: CCDI, C3DC, CTDC, ICDC, CDS, CCDI-DCC).

## Extending the framework

- **Add manual tests**: Add modules under `tests/test_manual/` and use the `api_client` fixture (and `test_data` when you need session discovery). `test_model_pvs_by_model.py` checks `GET /terms/model-pvs/{model}/` and `GET /terms/model-pvs/{model}/{property}` for each `MAJOR_MODELS` handle (by-model pin from `/versions`; property handle from session-cached aggregate candidates with non-empty `permissibleValues`, verified with property-level GET both omitting `version` and using `version=<versions[0]>`). Optional `version` query; PV rows may have null `ncit_concept_code` or empty `synonyms`. In `test_model_pvs_no_duplicates.py`, `test_model_pvs_no_duplicate_permissible_values` covers the same major models (discovery capped by `STS_DEDUP_LIMIT`, default **140** total split fairly across models — **20** props per model with **7** `MAJOR_MODELS`); `test_model_pvs_no_duplicates_bug_ticket_endpoints` pins specific model/property/version pairs from the original dedup bug ticket.
- **Null CDE manual tests**: `tests/test_manual/test_null_cde_all_models.py` is marked `nullcde`. The reference null-CDE **value** set is loaded once per session from `GET /terms/cde-pvs/16476366/1/pvs`, keeping only PV rows with **non-null** `ncit_concept_code` (NCIt-filtered). The **main** assertion: no data model at **latest** has a property whose PVs cover that full reference set, except **CDS** (latest contains `11.0.`) and **PSDC** (latest contains `1.0.`). A **second** check (parametrized) calls **CDS** pinned at `CDS_PINNED_VERSION` (e.g. **11.0.3**) and **PSDC** pinned at `PSDC_PINNED_VERSION` (pre-GA **1.0.0-5c0d1c3** until **1.0.0** exists in STS); each **passes** or **skips** (see module docstring). A **third** test hits `GET /terms/cde-pvs/11527735/1.00/pvs` with `use_null_cde` omitted, `false`, and `true`, using the same reference set for disjoint/subset checks. Run: `pytest -m nullcde tests/test_manual/test_null_cde_all_models.py -v` (uses `STS_BASE_URL`, default QA).
- **CDE-PVS legacy vs v2 manual tests**: `tests/test_manual/test_cde_pvs_legacy_vs_v2.py` is marked `cde_pvs_legacy`. Compares the **legacy** Data Hub–style endpoint `GET {origin}/cde-pvs/{id}/{version}?format=json` (origin = same host as v2 with `/v2` stripped via `sts_legacy_origin()`) to `GET /terms/cde-pvs/{id}/{version}/pvs` on the v2 client; asserts matching CDE metadata and that **each distinct** ``(value, ncit_concept_code)`` from legacy appears in v2 (legacy may duplicate rows; v2 may add extra rows). Run: `pytest tests/test_manual/test_cde_pvs_legacy_vs_v2.py -m cde_pvs_legacy -v` (uses `STS_BASE_URL`; no separate env var for legacy).
- **caDSR Designations vs STS PVS**: `tests/test_manual/test_cadsr_alternate_values_cdepvs_modelpvs.py` is marked `cadsr_alt_pvs`. Reads pinned CDE/model/property rows from [`data/cadsr_sts_pvs_cases.json`](data/cadsr_sts_pvs_cases.json); fetches caDSR `DataElement/{id}` (JSON includes `ValueDomain.PermissibleValues` and `Designations` under `ValueMeaning`) and asserts **every** ``Designations[].name`` (all types, e.g. ``MCL Alt Name`` is one kind) appears as a ``permissibleValues.value`` on both `GET /terms/cde-pvs/{id}/{version}/pvs` and `GET /terms/model-pvs/{model}/{property}?version=...`, with no duplicate PV ``value`` strings (case-sensitive). Optional ``CADSR_DESIGNATION_TYPES`` narrows by ``Designations[].type`` if you need a subset. Run: `pytest tests/test_manual/test_cadsr_alternate_values_cdepvs_modelpvs.py -m cadsr_alt_pvs -v`. Uses `STS_BASE_URL`, `CADSR_BASE_URL`, and optional `CADSR_DESIGNATION_TYPES` (see ONBOARDING §6.2).
- **Add unit tests**: Add modules under `tests/unit/` for runner logic with `APIResponse` mocks only (see `tests/unit/README.md`).
- **Adjust discovery**: Edit `src/sts_test_framework/discover.py` to change how test data is discovered.
- **Adjust generation**: Edit `src/sts_test_framework/generator.py` to add or change positive/negative cases (e.g. query param validation).
- **Contract validation**: Use the optional `runners.contract.run_contract_tests` and install `jsonschema` (and optionally `openapi-spec-validator`).
